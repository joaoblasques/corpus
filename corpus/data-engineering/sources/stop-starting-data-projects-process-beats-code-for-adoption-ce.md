# Stop Starting Data Projects — Process Beats Code for Adoption

> **Quick intake** (obsidian). raw stub: `notes-03-resources-articles-stop-starting-data-projects-stakeholder-process.md`

Data projects fail due to misaligned expectations rather than technical flaws. A process-oriented approach focusing on stakeholder psychology and adoption is crucial. This involves defining a 'definition of done' contract and delivering a rough MVP.
