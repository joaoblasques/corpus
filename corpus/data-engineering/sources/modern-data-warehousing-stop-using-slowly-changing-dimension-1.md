# "Modern Data Warehousing: Stop Using Slowly-Changing Dimensions (Part 1)"

> **Quick intake** (obsidian). raw stub: `notes-03-resources-articles-modern-data-warehousing-stop-using-scd-part-1.md`

This article discusses modern data warehousing, focusing on simplicity, speed, and resilience. It emphasizes the importance of date-stamping and the use of dimension tables and fact tables. The key takeaway is that storage is cheap, but developer time is not, and that a simple warehouse architecture can be more efficient than traditional approaches.
