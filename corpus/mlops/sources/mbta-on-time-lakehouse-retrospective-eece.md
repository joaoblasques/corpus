# mbta-on-time-lakehouse-retrospective

> **Quick intake** (obsidian). raw stub: `notes-06-metadata-reference-mbta-on-time-lakehouse-retrospective.md`

This document is a project retrospective on a lakehouse built to answer 'Is the train late, and why?' using MBTA GTFS-Realtime and GTFS static data. It covers lessons learned and reusable assets from the project. The key takeaway is the importance of self-managing loops and tiered autonomy in MLOps.

**Key topics**
- lakehouse
- MBTA GTFS-Realtime
- self-managing loops
- tiered autonomy
- MLOps
- Probe-then-Fallback design pattern
