# The Design of Approximation Algorithms (Williamson & Shmoys, 2011)

TL;DR: A graduate textbook by David P. Williamson and David B. Shmoys (Cambridge University Press, 2011; 500 pages; open-access electronic edition). Organized around **design techniques** rather than individual problems: each technique is applied to several NP-hard optimization problems. All 34 parts (Chapters 1-15, appendices, and index) fully ingested. Intended for graduate students and researchers; assumes undergraduate algorithms and NP-completeness background.

## Design techniques (Chapters 1-5)

### Greedy + set cover (Chapter 1)
The canonical greedy set cover algorithm: repeatedly pick the set that covers the most uncovered elements. Achieves an **H_n ≈ ln n** approximation ratio, where H_n is the n-th harmonic number. The proof uses the exchange-argument/potential technique: at each step, some set in the optimum must cover ≥ OPT/k remaining elements (for k = OPT size), so the greedy picks something at least that good. [^daa-p01]

### Submodular function maximization (Chapter 2)
A set function v is **submodular** if marginal gains diminish: v(X ∪ {ℓ}) − v(X) ≥ v(Y ∪ {ℓ}) − v(Y) whenever X ⊆ Y. The greedy algorithm (add the element with the highest marginal value) gives a **(1 − 1/e)** approximation for maximizing a monotone submodular function subject to a cardinality constraint. Proof via the inequality 1 − x ≤ e^{−x} applied repeatedly to the gap between greedy value and OPT. [^daa-p04]

### Bin packing (Chapter 3)
Bin packing is strongly NP-hard (no FPTAS unless P = NP). An **APTAS** (asymptotic polynomial-time approximation scheme) exists: for any ε > 0, there is a polynomial algorithm using at most (1+ε)OPT + O(1/ε²) bins. The approach rounds item sizes up to a bounded number of groups (eliminating tiny items via First Fit Decreasing), then uses LP rounding on the rounded instance.

### LP rounding for weighted vertex cover (Chapter 4)
LP relaxation: replace integer constraints x_v ∈ {0,1} with x_v ∈ [0,1]; solve LP in polynomial time. Round up: set x_v = 1 whenever x_v ≥ 1/2. This gives a **2-approximation**: every edge is covered (both LP constraints were ≥ 1/2, at least one rounds to 1), and the rounded solution costs ≤ 2 × OPT_LP ≤ 2 × OPT_IP.

### Randomized rounding — MAX SAT and MAX CUT (Chapter 5)
**MAX SAT**: set each variable to TRUE with probability 1/2 independently. Expected weight of satisfied clauses ≥ OPT/2 (since any clause of length ≥ 1 is satisfied with probability ≥ 1/2). If all clauses have length ≥ k, the guarantee improves to (1 − (1/2)^k) × OPT. The PCP theorem (Theorem 5.2) implies no (7/8 + ε)-approximation for MAX E3SAT unless P = NP. [^daa-p08]

**MAX CUT**: place each vertex in U independently with probability 1/2. Expected cut weight ≥ OPT/2. An edge is in the cut iff its endpoints are in different parts, which happens with probability 1/2 per edge. [^daa-p08]

## Key theorems

- **Greedy set cover**: H_k ≤ ln k + 1 ≈ ln(OPT) approximation ratio; tight — no better factor unless NP ⊆ DTIME(n^{O(log log n)}).
- **Submodular maximization**: (1 − 1/e) ≈ 0.632 approximation; tight for greedy.
- **(7/8 + ε)-hardness of MAX E3SAT**: if such an algorithm exists, P = NP (from the PCP theorem). [^daa-p08]

### Semidefinite programming rounding — MAX CUT (Chapter 6)
The **Goemans-Williamson algorithm** for MAX CUT achieves a **0.878-approximation** (improvement over the 1/2 randomized algorithm of Chapter 5). The SDP relaxation places each vertex as a unit vector in R^n; the objective is to maximize the sum of (1 − v_i · v_j)/2 over edges. Rounding: choose a random hyperplane through the origin; vertices on opposite sides go into different parts of the cut. The probability an edge (i, j) is cut = arccos(v_i · v_j)/π, which is ≥ 0.878 × (1 − v_i · v_j)/2 for all vector configurations. This was the first use of SDP in approximation algorithms. [^daa-p11]

**Unique games conjecture** (Khot 2002, stated in Chapter 6): If the Unique Games Conjecture is true, no polynomial-time algorithm can achieve a ratio better than 0.878 for MAX CUT. This conjecture has driven much of hardness-of-approximation research. [^daa-p11]

### Primal-dual method — Steiner tree, facility location (Chapter 7)
The primal-dual method builds a primal solution incrementally while raising dual variables. Key insight: increase dual variables corresponding to "active" constraints (constraints the current primal solution violates); when a constraint becomes tight, include its corresponding primal variable. This avoids solving an LP, yielding faster algorithms in practice. [^daa-p11]

**Generalized Steiner tree / Steiner forest (Section 7.4)**: Given a graph with edge costs and k terminal pairs (s_i, t_i), find minimum-cost subgraph connecting each pair. Primal-dual gives a **2-approximation** by growing moats around connected components simultaneously. [^daa-p11]

**Uncapacitated facility location (Section 7.6)**: facilities have opening costs f_j; serving client i from facility j costs c_{ij}. The primal-dual 3-approximation uses dual variables for clients (assignment costs) and raises them until a facility "opens" or a client is connected. [^daa-p11]

**k-Median (Section 7.7)**: identical to facility location but with k facilities; no opening costs. Lagrangean relaxation (introduce a penalty λ for each opened facility) reduces k-median to facility location; binary search on λ gives a (3+ε)-approximation. [^daa-p11]

### Cuts and metrics (Chapter 8)
**Multiway cut (Section 8.1)**: given k terminal vertices, remove minimum-weight edges so no path connects two terminals. LP relaxation assigns each edge a fractional length forming a semi-metric. Rounding: for each terminal, take the "Voronoi region" and cut its boundary; dropping one of the k cuts yields a **(2 − 2/k)-approximation**. [^daa-p14]

**Multicut (Section 8.2)**: given k source-sink pairs, separate all pairs. LP relaxation uses spreading metrics; rounding via region-growing gives an **O(log k)-approximation**. [^daa-p14]

**Sparsest cut (Section 8.3)**: minimize (edges crossing cut) / (demands separated). LP dual is a maximum-concurrent-flow problem. Solving the LP gives an O(log n)-approximation; SDP gives O(√(log n)) via Arora-Rao-Vazirani. [^daa-p15]

**Tree metrics (Section 8.4)**: any metric embeds into a distribution over tree metrics with expected distortion O(log n). This is the Bartal / Fakcharoenphol-Rao-Talwar result. Reduces many cut problems to their tree counterparts, enabling efficient approximations. [^daa-p15]

### Scheduling (Chapters 9-10)
**Scheduling on unrelated parallel machines (Chapter 10)**: jobs have machine-dependent processing times p_{ij}; minimize makespan. LP with time-indexed variables + LP rounding (assign job to machine where its fractional assignment is ≥ 1/2) gives a **2-approximation**. [^daa-p18]

### Survivable network design (Chapter 11)
**Survivable network design**: given connectivity requirements r_{ij} for each pair (i,j), find minimum-cost set of edges. The iterated rounding technique (solve LP, round edges whose LP value is ≥ 1/2, remove from LP, repeat) gives a **2-approximation**. Key lemma: in any extreme point solution of the LP, at least one variable is ≥ 1/2 (proved using the structure of "tight sets" and uncrossing). [^daa-p20]

### Prize-collecting problems (Chapter 14)
**Prize-collecting Steiner tree (Section 14.1)**: either connect a terminal or pay a penalty. Lagrangean relaxation + LP rounding gives a **(2 − 1/(n-1))-approximation** (Goemans-Williamson 1995). [^daa-p25]

### Hardness of approximation (Chapter 15)
**PCP theorem** (probabilistic checkable proofs): NP = PCP[O(log n), O(1)]. Meaning: any NP language has a proof verifier that uses O(log n) random bits and reads O(1) bits of the proof, accepting valid proofs always and rejecting invalid proofs with constant probability. This yields the **(7/8 + ε)-hardness of MAX E3SAT** and the **inapproximability of set cover** below (1 − 1/e) ln n (unless NP ⊂ DTIME(n^{O(log log n)})). [^daa-p26]

**Unique games conjecture** governs the tight ratio for many problems. The tight hardness of vertex cover remains open: the best known algorithm is a 2-approximation but a tight UGC-based hardness has not been fully proven. [^daa-p26]
