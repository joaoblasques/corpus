# 5 Simple Habits for Writing Clean Code

> **Quick intake** (obsidian). raw stub: `notes-03-resources-articles-5-simple-habits-for-writing-clean-code.md`

This document outlines five simple habits for writing clean code, including flattening logic, using clear and meaningful names, keeping functions focused, giving numbers a name, and commenting on the 'why' behind the code. By following these habits, developers can make their code easier to read, maintain, and trust. The document provides examples and best practices for implementing these habits in Python.
