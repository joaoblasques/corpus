# Gaussian Mixture Models

TL;DR: A GMM models a dataset as a weighted sum of K Gaussian components: p(x) = sum_k pi_k N(x | mu_k, Sigma_k). It captures multi-modal densities that a single Gaussian cannot. Parameters (means, covariances, mixing weights) are fitted by maximizing log-likelihood via the EM algorithm, which alternates between computing "responsibilities" (E-step) and updating parameters (M-step). EM guarantees non-decreasing likelihood at each iteration but converges only to a local optimum.

## Model Definition

A **Gaussian Mixture Model** with K components [^src1]:

```
p(x) = sum_{k=1}^K pi_k N(x | mu_k, Sigma_k)
```

where:
- **pi_k** = mixing weight (mixing coefficient) for component k, with pi_k >= 0 and sum_k pi_k = 1
- **mu_k** ∈ R^D = mean of component k
- **Sigma_k** ∈ R^{D x D} = covariance matrix of component k (symmetric positive definite)
- Each N(x | mu_k, Sigma_k) is a multivariate Gaussian density

**Parameters**: theta = {pi_k, mu_k, Sigma_k}_{k=1}^K. Total: K(1 + D + D(D+1)/2) - 1 parameters.

**Interpretation**: a data point x is generated by first selecting component k with probability pi_k, then drawing x from N(mu_k, Sigma_k).

## Latent Variable Perspective

Introduce a discrete latent variable z ∈ {1, ..., K} representing the component assignment [^src1]:

```
p(z = k) = pi_k
p(x | z = k) = N(x | mu_k, Sigma_k)
```

The joint: p(x, z) = p(z) p(x|z). Marginalizing z recovers the GMM: p(x) = sum_k p(z=k) p(x|z=k).

**Posterior (responsibility)**: given observed x, the probability that component k generated it is:

```
r_{nk} = p(z_n = k | x_n) = pi_k N(x_n | mu_k, Sigma_k) / p(x_n)
```

where p(x_n) = sum_j pi_j N(x_n | mu_j, Sigma_j). The r_{nk} are called **responsibilities** — they form a soft assignment of each data point to each component [^src1].

Properties: r_{nk} >= 0 and sum_k r_{nk} = 1 for each n.

## Log-Likelihood

For N i.i.d. observations {x_1, ..., x_N}, the log-likelihood is [^src1]:

```
L(theta) = sum_{n=1}^N log p(x_n | theta) = sum_{n=1}^N log [ sum_{k=1}^K pi_k N(x_n | mu_k, Sigma_k) ]
```

**Challenge**: the log is outside the sum — no closed-form MLE. Cannot solve for all parameters simultaneously because responsibilities r_{nk} depend on all parameters [^src1].

**Solution**: EM algorithm — iterative approach that converges to a local maximum.

## EM Algorithm

The **Expectation-Maximization (EM) algorithm** for GMMs alternates two steps [^src1]:

**E-step (Expectation)**: fix current parameters theta^{old}, compute responsibilities:

```
r_{nk} = pi_k N(x_n | mu_k, Sigma_k) / [ sum_j pi_j N(x_n | mu_j, Sigma_j) ]
```

This gives a "soft assignment" of each data point to each component. Define N_k = sum_n r_{nk} = total responsibility of component k.

**M-step (Maximization)**: fix responsibilities r_{nk}, update parameters to maximize expected complete-data log-likelihood:

```
mu_k^{new} = (1/N_k) sum_n r_{nk} x_n

Sigma_k^{new} = (1/N_k) sum_n r_{nk} (x_n - mu_k^{new})(x_n - mu_k^{new})^T

pi_k^{new} = N_k / N
```

**Intuition for mu update**: the new mean is a weighted average of all data points, where weights are the responsibilities. Data points for which component k has high responsibility pull the mean more strongly [^src1].

**Intuition for Sigma update**: the new covariance is a responsibility-weighted empirical covariance around the new mean.

**Intuition for pi update**: the new mixing weight is just the fraction of total responsibility attributable to component k.

## Convergence Properties

**Guaranteed**: the log-likelihood L(theta) is non-decreasing at every EM iteration [^src1].

**Not guaranteed**: convergence to the global maximum. EM converges to a local optimum or saddle point depending on initialization.

**Practical mitigation**:
- Multiple random restarts; keep the solution with highest final log-likelihood
- K-means initialization: use K-means cluster centers as initial mu_k, set Sigma_k = I, pi_k = 1/K
- Component collapse: if a component's responsibility N_k → 0, the covariance may become ill-conditioned; add regularization epsilon*I to Sigma_k

## Latent Variable EM Interpretation

EM maximizes the ELBO (Evidence Lower BOund), which is derived from Jensen's inequality applied to the log-likelihood [^src1]:

```
L(theta) >= E_{q(z)}[log p(x, z | theta) - log q(z)]
```

This interpretation generalizes beyond GMMs to variational inference. The E-step sets q(z) = p(z|x, theta^{old}) (optimal choice that makes bound tight); the M-step maximizes over theta.

## GMM vs. K-Means

| | GMM | K-Means |
|---|---|---|
| Assignment | Soft (probability per cluster) | Hard (one cluster) |
| Shape | Elliptical Gaussians (via Sigma_k) | Spherical (implicitly unit covariance) |
| Output | Full density estimate p(x) | Cluster centroids only |
| Initialization | Sensitive; multiple restarts needed | Sensitive; K-means++ mitigates |
| K-Means as special case | GMM with Sigma_k = epsilon*I as epsilon → 0 recovers K-means | — |

## Applications in ML

- **Density estimation**: model the distribution of training data for anomaly detection (low-probability regions = anomalies)
- **Soft clustering**: when cluster membership is uncertain, GMM provides probabilistic assignments
- **Speech and audio**: GMMs were standard in speech recognition (HMM-GMMs) before deep learning
- **Image segmentation**: model pixel distributions per segment
- **Generative model**: sample new data: draw k ~ Categorical(pi), then x ~ N(mu_k, Sigma_k)
- **Initialization for deep generative models**: GMMs can approximate the data distribution as a baseline

## Number of Components K

K is a hyperparameter. Model selection criteria:
- **BIC** (Bayesian Information Criterion): -2 log L + K * log N (penalizes complexity)
- **AIC** (Akaike Information Criterion): -2 log L + 2K
- **Cross-validation** on held-out log-likelihood
- **Dirichlet Process GMM**: nonparametric extension that infers K from data
