# "AI Agent Observability: How to Monitor Agents Running for Hours Without Babysitting"

> **Quick intake** (obsidian-list). [open source](https://www.mindstudio.ai/blog/ai-agent-observability-monitor-long-running-agents-3/)

This document covers AI agent observability, focusing on monitoring long-running AI agents to prevent failures and cost overruns. It outlines the importance of tracing costs, latency, failures, and decisions, and provides guidance on setting up distributed tracing for agent workflows. The key takeaway is to track four core signals: execution traces, cost tracking, latency profiling, and failure and anomaly detection.
