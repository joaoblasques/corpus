# Common Design Patterns

> **Quick intake** (obsidian). raw stub: `notes-03-resources-articles-common-design-patterns.md`

This document covers common design patterns in software engineering, including Singleton, Factory, Strategy, Observer, Decorator, and Adapter. It explains the purpose, use cases, and implementations of each pattern, along with their pros and cons. The key takeaway is that design patterns provide reusable solutions to common problems, promoting better code structure and flexibility.

**Key topics**
- Singleton Pattern
- Factory Method Pattern
- Strategy Pattern
- Observer Pattern
- Decorator Pattern
- Adapter Pattern
