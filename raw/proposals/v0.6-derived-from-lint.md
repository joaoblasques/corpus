## Exit code convention

No existing lint tooling exists in this repository — v0.5 lint is a manual/cognitive process (CLAUDE.md §8.3). The following convention is proposed for a future lint CLI. It draws from markdownlint and eslint conventions; no existing project standard was found to override.

| Exit code | Meaning |
|---|---|
| 0 | No findings — all rules pass |
| 1 | Warnings only — no errors (Rule 5 fired, no Rule 1–4 violations) |
| 2 | One or more errors (any of Rules 1–4 fired) |

Errors block action (e.g., a future pre-commit hook fails). Warnings are surfaced but non-blocking.

## Prerequisite: slug index

Every rule uses a shared slug index built before any rule runs.

```
function build_slug_index(corpus_root):
    index = {}  # bare_slug → "domain/bare_slug"

    for each file matching corpus_root/<domain>/<bare_slug>.md
        where <domain> does NOT start with "_"  # skip _config, _index, _log, _domains
        and <domain> is a directory (not a bare file):
        slug = filename.remove_suffix(".md")
        if slug in index:
            ABORT "Slug collision: '{slug}' found in both {index[slug]}
                   and {domain}/{slug} — v0.5 uniqueness invariant violated"
        index[slug] = "{domain}/{slug}"

    return index  # e.g. {"microservices": "software-engineering/microservices", ...}
```

Hub pages (`README.md`) are excluded from the index because hub slugs are the domain name, not a content page slug.

## Cross-rule execution order

When multiple rules fire on the same page, findings are reported in this order:

| Priority | Rule | Rationale |
|---|---|---|
| 1 | Rule 3 — self-reference | Cheapest check; structural error; must resolve before graph ops |
| 2 | Rule 1 — existence | Resolves all slug edges; broken edges make Rule 2 meaningless |
| 3 | Rule 2 — acyclicity | **Skip for pages with Rule 1 errors** — dangling edges produce spurious cycle reports |
| 4 | Rule 4 — source substitution | Independent of graph structure; always runs |
| 5 | Rule 5 — cardinality-1 | Warning; surfaced last |

**Rule 2 skip condition**: if Rule 1 reported one or more errors on any page P, exclude P's outbound edges from the graph before running Rule 2. Cycles involving P are deferred until its existence errors are fixed.

## Rule 1 — Existence

**Severity:** ERROR
**Exit code:** 2

### Algorithm

```
function check_existence(page, slug_index):
    findings = []
    page_id = "{page.domain}/{page.path.stem}"

    for slug in (page.frontmatter.derived_from or []):
        if slug not in slug_index:
            findings.append(ERROR, page_id,
                "derived_from: target '{slug}' not found in corpus")
    return findings
```

Each missing slug produces a separate finding. Do not collapse multiple missing targets into one message.

### Error message format

```
ERROR [{page_domain}/{page_slug}] derived_from: target '{slug}' not found in corpus
```

**Example:**
```
ERROR [software-engineering/distributed-systems-analysis] derived_from: target 'cap-theorem' not found in corpus
ERROR [software-engineering/distributed-systems-analysis] derived_from: target 'eventual-consistency' not found in corpus
```

## Rule 2 — Acyclicity

**Severity:** ERROR
**Exit code:** 2
**Prerequisite:** Rule 1 must pass for all pages in the derivation graph (or skip affected edges per the cross-rule order above).

### Algorithm

Three-color DFS. Colors: `WHITE` (unvisited), `GRAY` (in current DFS stack), `BLACK` (fully visited).

```
function build_derivation_graph(all_pages, slug_index, existence_errors):
    # existence_errors: set of page slugs that had Rule 1 violations
    graph = {}  # slug → [slug, ...]

    for page in all_pages:
        slug = page.path.stem
        if page.frontmatter.derived_from is None:
            graph[slug] = []
        else:
            # Drop edges whose source had existence errors, and drop dangling targets
            if slug in existence_errors:
                graph[slug] = []
            else:
                graph[slug] = [t for t in page.frontmatter.derived_from
                               if t in slug_index]
    return graph

function check_acyclicity(graph):
    color   = {slug: WHITE for slug in graph}
    stack   = []   # current DFS ancestor path
    findings = []

    function dfs(slug):
        color[slug] = GRAY
        stack.append(slug)
        for neighbor in graph.get(slug, []):
            if color.get(neighbor) == GRAY:
                # Cycle found: extract the cycle path from the stack
                cycle_start = stack.index(neighbor)
                cycle_path  = stack[cycle_start:] + [neighbor]
                findings.append(ERROR, scope="graph",
                    "derived_from: cycle detected: {' -> '.join(cycle_path)}")
            elif color.get(neighbor, WHITE) == WHITE:
                dfs(neighbor)
        stack.pop()
        color[slug] = BLACK

    for slug in graph:
        if color[slug] == WHITE:
            dfs(slug)

    return findings
```

### Error message format

Cycles span multiple pages; the error is reported at graph scope, not at a single page.

```
ERROR derived_from: cycle detected: {slug_A} -> {slug_B} -> ... -> {slug_A}
```

**Example:**
```
ERROR derived_from: cycle detected: cycle-node-alpha -> cycle-node-beta -> cycle-node-alpha
```

## Rule 3 — No self-reference

**Severity:** ERROR
**Exit code:** 2

### Algorithm

```
function check_self_reference(page):
    findings = []
    page_slug = page.path.stem
    page_id   = "{page.domain}/{page_slug}"

    if page_slug in (page.frontmatter.derived_from or []):
        findings.append(ERROR, page_id,
            "derived_from: page lists itself")
    return findings
```

### Error message format

```
ERROR [{page_domain}/{page_slug}] derived_from: page lists itself
```

## Rule 4 — No source substitution

**Severity:** ERROR
**Exit code:** 2

### Trigger condition

`derived_from:` non-empty **AND** `sources:` absent or empty **AND** `status: mature`.
<!-- Phase 2 review (a), 2026-05-25: `status: mature` confirmed valid
     per CLAUDE.md §4 frontmatter template (`status: stub | draft | mature`).
     Schema defines the token but not its semantics; graduation criteria
     for `mature` are a separate schema-doc concern, out of scope here. -->

This targets the specific case where a page claims full maturity while having zero external citations — only internal derivations. Pages with `status: stub` or `status: draft` and no sources are not errors; they may be legitimately under construction.

### Algorithm

```
function check_source_substitution(page):
    findings = []
    page_slug = page.path.stem
    page_id   = "{page.domain}/{page_slug}"

    has_derived = bool(page.frontmatter.derived_from)
    source_count = len(page.frontmatter.sources or [])
    is_mature    = (page.frontmatter.status == "mature")

    if has_derived and source_count == 0 and is_mature:
        findings.append(ERROR, page_id,
            "derived_from: status:mature but sources: absent or empty — "
            "derived_from entries do not count toward graduation")
    return findings
```

### Error message format

```
ERROR [{page_domain}/{page_slug}] derived_from: status:mature but sources: absent or empty — derived_from entries do not count toward graduation
```

**Example:**
```
ERROR [ai-engineering/retrieval-synthesis] derived_from: status:mature but sources: absent or empty — derived_from entries do not count toward graduation
```

### Scope note

`sources:` count for graduation is computed from the `sources:` list only. `derived_from:` entries are structurally ineligible as sources regardless of how many are present.

### Fixture

`fail-source-substitution.yaml` — `status: mature`, `sources:` absent, `derived_from:` with three real slugs.

## Rule 5 — Cardinality-1 warning

**Severity:** WARNING
**Exit code:** 1 (if no Rule 1–4 errors also present; otherwise the exit code is determined by the error rules)

### Algorithm

```
function check_cardinality(page):
    findings = []
    page_slug = page.path.stem
    page_id   = "{page.domain}/{page_slug}"
    derived   = page.frontmatter.derived_from or []

    if len(derived) == 1:
        findings.append(WARNING, page_id,
            "derived_from: single entry '{derived[0]}' — "
            "consider an inline wikilink instead")
    return findings
```

### Warning message format

```
WARN [{page_domain}/{page_slug}] derived_from: single entry '{slug}' — consider an inline wikilink instead
```

**Example:**
```
WARN [data-engineering/scd2-synthesis] derived_from: single entry 'scd2' — consider an inline wikilink instead
```

### Fixture

`warn-cardinality-1.yaml` — `derived_from:` with exactly one entry.
