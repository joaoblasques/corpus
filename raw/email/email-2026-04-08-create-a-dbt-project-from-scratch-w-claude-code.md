Using AI tools just for the sake of using AI isn't very
helpful.

If anything, it'll just help you hurry up to make a mess.

But of course there are great use cases for us data folks that we
shouldn't ignore.

For example, working with code base tools like dbt.

So in today's video, I'll walk through a simple example of how
you can create a new dbt project from scratch, using Claude Code.

We'll cover things like:

* Using the CLAUDE.md file to clarify the project purpose &
design
* How uploading a sample dbt project can help the AI and get you
a better result
* Creating "rules" to continuously improve long-term

To be clear, this doesn't mean that you'll never have to review
code or double check the work.

Blindly trust the AI outputs at your own risk.

But if you want to expedite some of the grunt work, this will
help you get from zero to one a lot faster.

And help set foundations for a well structured, clean project
going forward.
