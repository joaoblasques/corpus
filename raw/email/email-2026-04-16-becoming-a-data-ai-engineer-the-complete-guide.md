Article Series
In Part 1 I mentioned that RAG is the ETL of AI, that security is an architecture decision, and that MCP changed how I work across the full stack. These three series are the deep dives behind those claims.
RAG for Data Engineers (4 parts)
Start here: LLM Limitations, Fundamentals, Alternatives [ https://substack.com/redirect/e5dfc591-488d-4a10-83e3-1b394c99ace5?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
Four articles covering why RAG exists, vector databases and embeddings, similarity search with metadata filters, and a full hands-on implementation with OpenAI, Agno and PgVector. The series ends with a working pipeline you can clone from the companion repo [ https://substack.com/redirect/5b52245e-1eb4-44ef-ac8b-c571d5cb4c23?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ].
The data engineering part defines 80% of the project. Don’t rush it. Craft it properly.
Only fine-tune if prompting and RAG fall short. 100 precise examples outperform 1,000 mediocre ones.
Introduction To Fine Tuning Pipelines For Data Engineers [ https://substack.com/redirect/3120a218-1356-4d28-9b8d-81793c7d589a?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
Secure LLMs for Data Engineers (4 parts)
Start here: Structured Outputs [ https://substack.com/redirect/445b4d5c-7c4b-4f5a-b130-69eeff6b7de2?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
Four articles: structured outputs for reliability, prompt injection prevention, secure design principles (least privilege, scoped tools), and a hands-on guardrails implementation with Agno. Clone the ShamanAI repo [ https://substack.com/redirect/e78bc4ea-5c78-456a-833a-35c0766bcd0a?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ] for working examples of all four security layers.
Scope your agent’s permissions at the architecture stage. An agent can’t drop a production table it doesn’t have access to.
MCP & SKILLs (4 parts)
Start here: He Built an MCP Server So You Can Talk to AI About Any Substack Author [ https://substack.com/redirect/e9e3eba7-dc15-4b1e-8c4a-ca28f8e484dc?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
The origin story, then the architecture breakdown (Behind Substack Author MCP [ https://substack.com/redirect/c9466bd1-0518-49a8-a77a-d60e050589e7?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]), the concept of SKILLs as progressive disclosure for agents (What’s the real deal about SKILLs [ https://substack.com/redirect/bccf1fe1-7535-44d1-b1f8-ad00a898e183?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]), and a full hands-on build (Building a Substack Agent with SKILLs and MCP [ https://substack.com/redirect/b18f2fac-d9e5-4683-9e65-930a0a0be5ac?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]). The final article ships a working agent with 5 SKILLs and remote MCP tools.
SKILLs encode the judgment for using tools. Publishing them through MCP makes that judgment available to every connected client.

Production Lessons
The instinct gap from Part 1: AI identifies the error but suggests removing Pandas entirely. These articles cover what actually breaks in production and how to build the judgment layer around it.
Learnings From 1 Year of Building AI Agents [ https://substack.com/redirect/aa6f1812-a56c-41fa-b576-2f04a56cfeb6?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
Less system prompt equals more consistency. Tool design is critical because each description is a prompt itself.
Had a 1,000-line prompt. Cut it to 250. Users said “now it gets what you want.” Less is genuinely more.
The Untold Pains About Building AI Agents On Production [ https://substack.com/redirect/f7d1ebac-c30a-496d-98b7-2ab496ec5beb?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
pgvector is probably all you need for RAG. Model updates will break your agent. Pin versions, always have a rollback plan.
How AI Can Improve Your Data Modeling (And What’s Still Missing) [ https://substack.com/redirect/1220dbce-ae82-4a15-b872-d85664bba41c?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ]
The article that started the conversation. AI keeps improving but data development workflows are still missing real AI-assisted context.

Data Engineers Are Becoming MetadataOps Engineers (And You Don’t Even Know It Yet) [ https://substack.com/redirect/40f17039-16c1-4378-bce0-9cc6fb099719?j=eyJ1IjoibjBrMyJ9.SQ5ZKHxJh0MocVfrsArKZFhT_ts26OEfWqtevr5N8ms ] — with Data Gibberish
