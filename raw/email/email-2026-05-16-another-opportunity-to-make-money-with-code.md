Hey Joao,

Last week, I was at an event about AI tools for small businesses.

Most of the conversation was what you would expect.

People were asking about ChatGPT, automation tools, agents, and all the new stuff that everyone is trying to figure out right now.

But during the Q&A, someone brought up a problem their team was still dealing with every morning.

It was not some crazy technical problem.

It was simple.

Their team had to log into a few different web portals every day, check the status of a few things, copy the updates into a spreadsheet, and then send a short report to the rest of the team.

And the funny thing is, nobody in the room reacted like this was some huge problem.

Because on the surface, it is not.

It is boring.

But that is exactly what made it interesting to me.

Because a lot of real opportunities for developers do not start with a huge app idea.

They start with a boring workflow someone is already tired of doing by hand.

This is the kind of thing a lot of beginners miss.

They think coding only becomes valuable once they can build a big app, pass a hard interview, or work at a major tech company.

But in the real world, companies lose time in much smaller ways.

A team might use software all day, but still copy data between tools, check the same dashboard every morning, or send the same update manually because the tools do not connect cleanly.

And that is where the opportunity is.

In that example from the event, the first version of the solution does not need to be some massive AI product.

It could be a small browser automation workflow that handles the repeatable parts: open the portals, check the statuses, write the updates into a sheet, and send a short summary.

Then save a few logs or screenshots so someone can verify what happened.

And keep a human in the loop for anything weird.

That is the key.

Use code for the stable parts.

Use AI only when the page is messy or changes often.

And let a human review anything important before it gets submitted, changed, paid, deleted, or sent.

The goal is not to make an AI agent rethink the whole process from scratch every morning, but to turn the repeatable parts into something that runs the same way every time.

REMEMBER: companies do not always need a brand-new app.

Sometimes they need someone who can look at a messy workflow and remove the manual steps.

That kind of problem can become a portfolio project.

It can become a freelance offer.

It can become a strong interview story.

And it can become proof that you know how to use code to solve a real business problem.

Talk soon,
