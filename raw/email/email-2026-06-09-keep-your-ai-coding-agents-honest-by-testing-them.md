Steve Kinney on using Playwright and tests to verify what your agents actually shipped.

Hi jonas,

Playwright drives a real browser the way a user would: clicking, typing, waiting for things to load, then asserting the app did what it should. That makes it the perfect reality check for AI coding agents. An agent can claim a feature works, but a passing Playwright test is proof.

-----------------------------------------------------------------------------------------------------------------------------------------------------------
Playwright: Automated Testing & AI Workflows ( https://master.dev/courses/playwright/?utm_source=course-announce&utm_medium=email&utm_campaign=playwright )
-----------------------------------------------------------------------------------------------------------------------------------------------------------

In this course, Steve Kinney walks through building the infrastructure that keeps coding agents honest: end-to-end tests with Playwright, deterministic network mocking, visual regression checks, and agent loops that won't call something done before it actually is.

Steve Kinney teaching his new Playright course ( https://master.dev/courses/playwright/?utm_source=course-announce&utm_medium=email&utm_campaign=playwright )

*What you'll walk away with:*

* Write end-to-end Playwright tests that give agents real feedback to act on.
* Reuse authentication and browser state so your test suite runs fast and stays stable.
* Mock network requests with HAR files and route interception for repeatable results.
* Catch unexpected UI changes automatically with visual regression testing.
* Set up agent loops and git hooks that enforce quality before code hits the repo.

Your real job was never typing code, it was architecting systems you can be confident in. This course hands you the tools to build that confidence into your agentic workflow from day one.
